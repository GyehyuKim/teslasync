# TeslaSync 디자인 시스템

폰에서 대시캠 클립을 빠르게 훑고 받는 **유틸리티 UI**를 위한 디자인 기준.
웹 UI(nginx/fancyindex)와 향후 Android 앱이 이 문서를 공통 기준으로 삼는다.

## 1. 원칙

- **유틸리티 우선(App UI, not marketing).** 차분한 표면, 강한 타이포, 색은 최소.
  꾸밈보다 명료함. "삭제해서 나아지면 계속 삭제한다."
- **모바일 우선.** 주 사용처는 폰 브라우저(보드 WiFi 접속 후). 작게 설계하고 위로 확장.
- **다크 기본.** 차 안·어두운 환경에서 영상 확인. 순수 검정 대신 **밝기로 층위(elevation)**를 준다.
- **가볍게.** 임베디드 보드에서 도는 정적 페이지. 웹폰트·프레임워크 없이 즉시 뜬다.
- **누르는 건 눌리게.** 터치 타깃 44px+, hover에 의존하지 않는다(모바일엔 hover 없음).

## 레퍼런스(북극성)
Linear(다크 유틸리티) · Jellyfin/Plex(미디어 브라우징) · iOS Files/Photos(모바일
파일·미디어) · GitHub/Vercel 다크모드. 공통점: **차분한 표면 + 하나의 강조색 + 밀도
높지만 읽히는 리스트.**

## 2. 색 (Design tokens)

```css
:root {
  /* 표면 — 밝기로 층위 (순수 검정 지양) */
  --bg:            #16181d;  /* 페이지 배경 */
  --surface:       #1e2127;  /* 리스트 행·카드 */
  --surface-hover: #272b32;  /* 행 hover/press */
  --elevated:      #22262d;  /* sticky 헤더 등 떠 있는 면 */
  --border:        #2c313a;  /* 구분선 */
  --border-subtle: #23272e;  /* 행 사이 얇은 선 */

  /* 텍스트 — 순수 흰색 지양(눈부심) */
  --text:      #e8eaed;  /* 본문 (대비 ~13:1) */
  --text-2:    #9aa0a6;  /* 보조: 용량·날짜·라벨 (대비 ~6.4:1) */
  --text-3:    #6b7280;  /* 흐림: 상위폴더·비활성 */

  /* 강조 — 하나의 파랑 계열만 */
  --accent:       #7ab7ff;  /* 링크·주요 액션 (대비 ~8:1) */
  --accent-hover: #a8d0ff;
  --accent-tint:  rgba(122,183,255,0.12);  /* 선택·배경 틴트 */

  /* 의미색 */
  --success: #4ade80;  /* 녹화중·다운로드 완료 */
  --warning: #fbbf24;  /* 저장공간 경고 */
  --danger:  #f87171;  /* 삭제·오류 */
}
```

규칙: 비회색 색상은 **강조 파랑 + 의미색 3개**로 제한. 보라/청보라 그라데이션 금지.

## 3. 타이포그래피

```css
:root {
  /* 시스템 폰트 — 임베디드·로컬 전용이라 웹폰트 로딩 0, 네이티브 느낌, 지연 0.
     (마케팅 페이지면 system-ui는 '타이포 포기' 신호지만, 유틸리티 앱에선 정답) */
  --font: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
  --font-mono: ui-monospace, "SF Mono", Menlo, monospace;  /* 용량·시각: tabular-nums */

  /* 스케일 (모바일 기준, 본문 16px 이하로 내리지 않음) */
  --text-xs:   12px;  /* 라벨·메타 */
  --text-sm:   14px;
  --text-base: 16px;  /* 본문 최소 */
  --text-lg:   18px;
  --text-xl:   22px;  /* 페이지 제목 */
}
```

- 줄간격: 본문 1.5, 제목 1.25
- 굵기: 본문 400, 제목·라벨 600 (굵기 대비로 위계)
- 숫자 정렬 열(용량 등)엔 `font-variant-numeric: tabular-nums`
- 라벨은 소문자에 letter-spacing 금지, 헤더 라벨은 `text-transform:uppercase` + 미세 letter-spacing 허용

## 4. 간격·레이아웃

- **4px 스케일**: 4 · 8 · 12 · 16 · 24 · 32 · 48. 임의값 금지.
- **터치 타깃 최소 44px** (행 전체를 탭 영역으로 — 텍스트 줄만 눌리게 두지 않는다).
- 반경: `--r-sm: 6px`(버튼) · `--r-md: 10px`(카드). 전부 똑같은 큰 라운드(버블) 금지.
- 브레이크포인트(모바일 우선): 기본 `<480` · 태블릿 `481–768` · 데스크톱 `>768`
- 최대 본문 폭 지정(데스크톱에서 풀블리드 텍스트 금지), 가로 스크롤 0.
- `env(safe-area-inset-*)`로 노치 대응.

## 5. 컴포넌트

- **리스트 행(클립/파일):** 행 전체가 탭 영역(≥44px). 파일명(본문) + 용량·시각(보조,
  mono·tabular, 우측). hover 시 `--surface-hover`. 긴 파일명은 줄바꿈 허용.
- **헤더:** `position:sticky; top:0`, `--elevated` 면. 소형 대문자 라벨.
- **이벤트 카드(향후 클립 브라우저):** 썸네일(`thumb.png`) + 시각 + 사유/장소(`event.json`)
  + 카메라 수. 카드는 "카드가 곧 상호작용일 때만" 쓴다(장식용 카드 그리드 금지).
- **버튼:** 주요=accent 배경 / 보조=surface+border. 최소 44px. 라벨은 구체적으로
  ("이 클립 받기" — "확인" 금지).
- **상태:** 로딩=실제 레이아웃 닮은 스켈레톤 / 빈 상태=한 줄 안내+아이콘+액션
  ("없음" 금지) / 오류=무엇이·왜·다음 뭐 할지 / 녹화중=`--success` 점.

## 6. 인터랙션·모션

- 지속 120–200ms, 진입 ease-out. `transform`·`opacity`만 애니메이트(레이아웃 속성 금지).
- `transition: all` 금지 — 속성 명시.
- `prefers-reduced-motion: reduce` 존중.
- focus-visible 링 항상(‌`outline:none`만 두지 않는다).

## 7. 접근성

- WCAG AA: 본문 ≥4.5:1(현재 ~13:1), 링크 ~8:1, 보조텍스트 ~6.4:1 — 전부 통과.
- 색만으로 정보 표현 금지(아이콘·라벨 병행).
- 터치 44px, `cursor:pointer`는 클릭 가능한 것에만.
- 플레이스홀더를 유일한 라벨로 쓰지 않는다.

## 8. 피해야 할 것 (AI-slop / 유틸리티에 안 맞는 것)

- 보라·청보라 그라데이션 배경
- 아이콘-원-3열 피처 그리드, 전부 가운데 정렬, 전부 똑같은 큰 라운드
- 장식용 blob·물결 divider, 이모지를 UI 요소로, 카드 좌측 컬러 보더
- 본문 16px 미만, hover로만 발견되는 버튼(모바일)
- 제네릭 히어로 문구("Welcome to…") — 유틸리티 앱엔 히어로 자체가 불필요

## 현재 상태 → 목표

- **웹 UI(fancyindex):** 이미 다크테마·44px 터치타깃·WCAG 대비 적용됨. 이 문서의
  토큰으로 색·간격·타이포를 정식화하고, **RecentClips까지 나열**하도록 확장.
- **이벤트 카드 뷰:** 파일 나열 대신 이벤트 단위(썸네일+시각+사유) 카드로 개선.
- **Android 앱:** 위 토큰·컴포넌트 규칙을 그대로 따라 웹과 일관성 유지.
